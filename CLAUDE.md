# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

This is an **Obsidian vault** for personal knowledge management, focused on AI topics (Polish: "Sztuczna inteligencja" = "Artificial Intelligence"). It is not a software project — there are no build commands, tests, or source code to compile.

## Vault structure

Top-level folders represent topic areas:
- `Al-nowości i nie tylko` — AI news and general notes
- `Linki` — saved links
- `Trading` — trading-related notes
- `Youtube` — notes clipped from YouTube videos
- `Zdrowie` — health notes
- `copilot/` — AI assistant plugin data (conversations, custom prompts, memory, system prompts)

## Note conventions

Notes use YAML frontmatter for metadata. YouTube notes follow this schema (generated by the "Clip YouTube Transcript" custom prompt):

```yaml
---
title: "<video title>"
description: "<first 200 chars>"
channel: "<channel name>"
url: "<video url>"
duration: "<duration>"
published: YYYY-MM-DD
thumbnailUrl: "https://i.ytimg.com/vi/VIDEO_ID/maxresdefault.jpg"
genre:
  - "<genre>"
watched:
---
```

YouTube notes always include: embedded video link, collapsible Description callout, Summary, Key Takeaways, a Mermaid mindmap, and Notable Quotes with timestamps.

**Mermaid mindmap rules** (critical — Obsidian is strict):
- Root node: `root(Topic Name)` — round brackets only, never double brackets
- Child nodes: plain text only, no brackets, no quotes, no special characters, no emojis
- Max 3–4 words per node

## Key plugins

- **obsidian-git** (v2.38.2) — automatic Git backup and version control
- **copilot** — AI chat assistant; default model is `google/gemini-2.5-flash` via OpenRouter; embedding via `openai/text-embedding-3-small` via OpenRouter; autonomous agent enabled with tools: localSearch, readNote, webSearch, youtubeTranscription, writeFile, editFile, updateMemory
- **remotely-save** — cloud sync (separate from Git)
- **smart-connections** — AI-powered note linking; embeddings cached under `.smart-env/`
- **quickadd** — macro-based note capture
- **periodic-notes** — daily/weekly notes
- **omnisearch** — full-text vault search

## Copilot custom prompts

Located in `copilot/copilot-custom-prompts/`. Each file is a Markdown note with frontmatter keys that control Copilot behavior (`copilot-command-slash-enabled`, `copilot-command-model-key`, etc.). Adding a new custom prompt means creating a `.md` file in that folder with appropriate frontmatter.

Copilot conversations are auto-saved to `copilot/copilot-conversations/` with the naming pattern `{$topic}@{$date}_{$time}`.

## Working with this vault via Claude Code

Since this is a notes vault, typical tasks are:
- Creating or editing Markdown notes following the conventions above
- Updating custom Copilot prompt files in `copilot/copilot-custom-prompts/`
- Reading/summarizing existing notes
- Reorganizing files across topic folders

There are no lint, build, or test commands.
